# 🔧 技术负责人工作流程

## 我的职责（3 件事）

### 1️⃣ 把需求拆成可执行任务
- 扫描所有源码 → 找出 bug
- 按优先级分类 (P0/P1/P2)
- 生成标准化任务文档 (给 Cursor 用)

### 2️⃣ 把任务发给 Cursor 执行
- 每个任务都用固定格式:
  ```
  【项目】项目名
  【文件】要改的文件
  【问题】具体是什么问题
  【修改要求】怎么改
  【限制】不能做什么
  【验收标准】怎么验证
  ```
- 一次一个任务，Cursor 完成后我审核
- 不要一次性发多个任务

### 3️⃣ 审核 Cursor 的改动
**审核标准**:
- ✅ 不破坏现有钱包流程
- ✅ 不增加新的全局污染
- ✅ 不做无关重构
- ✅ 改动尽量小
- ✅ 改完能跑基础检查

## 核心原则

1. **一次一个任务** — 不要贪多，确保质量
2. **验收清单必须跑** — 改完代码就立即验证
3. **有新风险立即汇报** — 不隐瞒
4. **改动要尽量小** — 不做不必要的重构
5. **钱包流程优先** — 安全 > 稳定性 > 性能

---

## 问题排查

### 如果某个任务卡住了怎么办？

1. 检查限制条件（看【限制】部分）
2. 检查验收标准（看【验收标准】部分）
3. 如果需要改其他文件，提醒我
4. 必要时降低优先级，先做 P1

### 如果改动影响了现有流程怎么办？

1. **立即停止**，回滚改动
2. 详细记录症状
3. 向我汇报，分析根因
4. 重新设计改动方案

### 如果发现新的 bug 怎么办？

1. 记录到 `bugs/bugs.json` （加入新条目）
2. 评估优先级
3. 如果是 CRITICAL，插队到当前任务前
4. 向我汇报
