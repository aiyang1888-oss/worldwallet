# BUG-007 - 验证助记词页面刷新后，验证词随机重新抽取

**发现时间：** 2026-04-07 01:16  
**严重级别：** P1（影响用户体验，可能导致无法完成验证）

## 问题

在"验证助记词"页面，每次刷新或重新进入时，随机抽取的验证词会重新生成。

**复现步骤：**
1. 进入"我的密钥"页面，生成助记词
2. 点击"我已抄写，开始验证"
3. 进入验证页面，显示需要输入第 5、11、12 词
4. 刷新页面或返回后再进入验证页面
5. 验证词变成了第 6、7、8 词（或其他随机组合）

**预期行为：**
- 验证词应该在用户首次进入验证页面时确定，并在整个会话期间保持不变
- 或者：验证词保存在 localStorage/sessionStorage 中，刷新后仍然一致

**实际行为：**
- 验证词每次进入都随机重新抽取
- 用户如果忘记验证词编号，刷新后无法继续验证

## 更严重的关联问题

**BUG-007a：返回助记词页面会生成新钱包**

如果用户从验证页面返回到"我的密钥"页面（想要重新查看助记词），会触发 **新钱包生成**，导致：
1. 原来的助记词丢失
2. 用户无法完成验证
3. 原钱包的地址和私钥都变了

**这是一个灾难性的 UX bug！**

## 根因分析

可能的原因：

1. **验证词未持久化：**
   - `startVerify()` 函数每次调用都重新随机抽取验证词
   - 没有保存到 localStorage/sessionStorage

2. **助记词页面每次进入都重新生成钱包：**
   - `goTo('page-key')` 函数中可能有 `createRealWallet()` 调用
   - 应该检查当前会话是否已有钱包，只在没有时才生成新的

## 修复方案

### 1. 验证词持久化

在 `startVerify()` 函数中，将验证词编号保存到 `sessionStorage`：

```javascript
function startVerify() {
  const indices = selectRandomWords();
  sessionStorage.setItem('ww_verify_indices', JSON.stringify(indices));
  // 渲染验证页面...
}
```

在验证页面加载时，读取保存的编号：

```javascript
function loadVerifyPage() {
  let indices = sessionStorage.getItem('ww_verify_indices');
  if (indices) {
    indices = JSON.parse(indices);
  } else {
    indices = selectRandomWords();
    sessionStorage.setItem('ww_verify_indices', JSON.stringify(indices));
  }
  // 使用 indices 渲染验证输入框...
}
```

### 2. 防止重复生成钱包

在 `goTo('page-key')` 中，添加检查：

```javascript
if(pageId==='page-key') {
  // 检查是否已经有钱包在本会话中生成
  if (!window._currentWallet || !window._currentWalletMnemonic) {
    showWalletLoading();
    createRealWallet(currentMnemonicLength).then(...);
  } else {
    // 重新渲染现有钱包的助记词
    renderKeyGrid();
  }
}
```

### 3. 添加用户提示

在验证页面添加返回按钮的警告：

```
⚠️ 返回将重新生成钱包，请确保已抄写助记词
```

## 文件路径

- `dist/wallet.ui.js` - `startVerify()` 函数
- `dist/wallet.ui.js` - `goTo('page-key')` 逻辑
- `dist/wallet.html` - 验证页面 HTML

- 所有用户的钱包创建流程
- 用户可能因刷新或返回而丢失助记词
- 严重影响用户信任
