## 23:18 - 浏览器测试发现 P0 UX Bug

老郭要求：直接打开浏览器测试，不要问我，自己找 bug

### Bug 现象

访问 https://www.worldtoken.cc/wallet.html 时：
1. 页面加载后自动显示"正在生成钱包..."加载动画
2. 同时弹出 5 个对话框（PIN 输入、设置 PIN、Authenticator、绑定 Authenticator、万语地址弹窗）
3. 所有弹窗叠加，用户无法正常使用

### 根因分析

**技术细节：**
- 所有弹窗元素（`#walletLoadingOverlay`, `#pinModal`, `#pinSetModal`, `#totpModal`, `#totpSetModal` 等）
- CSS 中已正确设置 `display: none`（line ~865 in wallet.css）
- 但浏览器 computed style 显示为 `display: flex`
- 说明 JS 代码在页面加载时设置了内联样式

**正确逻辑应该是：**
- CSS 默认所有弹窗为 `display: none`
- JS 通过添加 `.show` class 或设置 `style.display` 来显示弹窗
- 只有用户主动触发操作（点击"创建钱包"等）时才显示对应弹窗

**可能的原因：**
1. 某个初始化函数（如 `showWalletLoading()`）被页面加载时自动调用
2. DOMContentLoaded 或 window.load 事件监听器中包含了显示弹窗的代码
3. HTML 中有内联 `<script>` 标签在加载时立即执行了显示逻辑


**问题分析：**
- 页面加载时调用了 `loadWallet()` 等初始化函数

### 修复方案

在上述代码块之后立即添加强制隐藏所有弹窗的逻辑：

```javascript
// ─── BUG-001 修复：强制隐藏所有弹窗（防止页面加载时自动显示） ────
try {
  const modalIds = [
    'walletLoadingOverlay',
    'pinModal',
    'pinSetModal',
    'totpModal',
    'totpSetModal',
    'worldAddressModal'
  ];
  
  modalIds.forEach(function(id) {
    const el = document.getElementById(id);
    if (el) {
      el.classList.remove('show');
      el.style.display = 'none';
    }
  });
} catch (_modalFix) {
  console.error('[BUG-001] 弹窗隐藏失败:', _modalFix);
}
// ────────────────────────────────────────────────────────────────
```
