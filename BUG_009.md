# BUG-009 - 地址页面文字颜色错误导致黑屏

**发现时间：** 2026-04-07 01:34  
**严重级别：** P1（阻塞核心功能）

## 问题

点击"地址"标签页后，页面内容不可见（黑屏）。

**复现步骤：**
1. 进入钱包首页
2. 点击底部导航栏的"地址"标签
3. 页面显示黑屏

**预期结果：** 显示万语地址、链地址列表等内容

**实际结果：** 页面黑屏，无法看到任何内容

## 根因分析

通过浏览器调试发现：

```javascript
{
  "backgroundColor": "rgb(6, 6, 15)",  // 深色背景
  "color": "rgb(0, 0, 0)"              // 黑色文字 ❌
}
```

**问题：** `.page` 或 `#page-addr` 的 CSS 规则将文字颜色设置为黑色（`color: #000` 或 `color: rgb(0,0,0)`），导致深色主题下文字不可见。

**正确的样式应该是：**
```css
.page {
  color: var(--text);  /* 或者不设置，继承 body 的颜色 */
}
```

## 修复方案

### 选项 1：移除 `.page` 的 `color` 样式

在 `wallet-shell/wallet.css` 中查找：
```css
.page {
  /* ... */
  color: #000; /* 或 color: rgb(0,0,0); */
}
```

移除 `color` 属性，让文字颜色继承自 `body` 或使用 CSS 变量。

### 选项 2：使用 CSS 变量

```css
.page {
  color: var(--text);  /* 深色主题是白色，浅色主题是黑色 */
}
```

### 选项 3：检查是否有 `[data-theme="light"]` 覆盖

确保深色主题下没有错误的颜色覆盖：
```css
[data-theme="light"] .page {
  color: #000;  /* 仅浅色主题使用黑色 */
}

/* 深色主题（默认）不需要覆盖 */
```

## 影响范围

- 所有使用 `.page` class 的页面可能都有同样的问题
- 需要测试其他页面（兑换、礼物、设置等）

## 文件路径

- CSS: `wallet-shell/wallet.css` 或 `dist/wallet.css`
- 可能涉及的 CSS 规则：`.page`, `#page-addr`, `.addr-content`
