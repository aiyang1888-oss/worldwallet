# 多语言地名词库规范

**版本:** 1.0  
**日期:** 2026-04-07  
**状态:** 最终确认

---

## 统一标准

**所有语言的 2048-词库统一规范：该语言的 2048 个常用地名**

### 核心原则

1. **地名优先** — 用真实的地理位置名称
2. **文化适配** — 选择该文化圈最著名、最易记忆的地名
3. **唯一性** — 每个地名独一无二，易于区分
4. **人类记忆** — 地名比抽象词汇更容易记忆和传达
5. **1:1 映射** — 与 BIP39 的 2048 词严格一一对应

## 词库创建流程

### Step 1：收集地名数据

**数据源：**
- 维基百科（各语言版本）
- 联合国地名数据库
- 谷歌地图（多语言）
- 当地政府或旅游局网站

### Step 2：整理为 JSON 数组

**格式：**
```json
[
  "城市1",
  "城市2",
  "城市3",
  ...
  "城市2048"
]
```

**验证清单：**
- ✅ 共 2048 个元素
- ✅ 每个元素是该语言的地名
- ✅ JSON 格式正确
- ✅ 无重复
- ✅ 无空值

### Step 3：验证与 BIP39 对应关系

**重要：** 词库的顺序必须与 BIP39 英文词表的顺序一一对应

例如：
```
BIP39 英文位置 0: "abandon" → 中文位置 0: "北京"
BIP39 英文位置 1: "ability" → 中文位置 1: "上海"
BIP39 英文位置 2: "able"    → 中文位置 2: "广州"
...
```

### Step 4：放置文件

```
wallet-shell/wordlists/
├── en.json     (BIP39 标准)
├── zh-cn.json  (中文简体)
├── zh-tw.json  (中文繁体)
├── ja.json     (日文)
├── es.json     (西班牙文)
├── fr.json     (法文)
├── de.json     (德文)
├── ru.json     (俄文)
├── pt.json     (葡萄牙文)
├── ar.json     (阿拉伯文)
├── hi.json     (印地文)
├── ko.json     (韩文)
└── th.json     (泰文)
```

## 总结

**统一标准：所有语言都使用 2048 个常用地名**

- ✅ 中文 → 中国地名
- ✅ 日文 → 日语翻译的地名
- ✅ 其他语言 → 该语言的地名

**核心优势：**
1. 人类记忆 — 地名比抽象词更容易记忆
2. 文化认同 — 用户对地名有现实感受
3. 国际通用 — 地名在所有文化中通用
4. 易于扩展 — 统一标准便于添加新语言
5. 安全可靠 — 与 BIP39 完全兼容
