# BUG-006 - 验证助记词页面：`fill` 工具无法输入内容

**发现时间：** 2026-04-07 01:15  
**严重级别：** P2（影响自动化测试，但不影响用户手动使用）

## 问题

在"验证助记词"页面（`#page-key-verify`），使用 OpenClaw `browser` 工具的 `fill` 操作无法向输入框输入内容。

**测试步骤：**
1. 进入"验证助记词"页面
2. 使用 `browser.act(kind='fill', fields=[...])`  填写3个输入框
3. 检查输入框的 `value` 属性

**预期结果：** 输入框显示填写的内容

**实际结果：** 输入框的 `value` 全是空字符串，placeholder 仍然显示

## 根因分析

可能原因：
1. 输入框有特殊的 event listener 阻止了 `fill` 操作
2. 需要触发特定的事件（`input` / `change` / `keyup` 等）才能让值生效
3. 输入框可能有 Vue/React 等框架的双向绑定，需要通过特定方式更新

## 临时解决方案

使用 `browser.act(kind='evaluate', fn=...)` 直接用 JS 设置值：

```javascript
() => {
  const inputs = document.querySelectorAll('#verifyQuestions input');
  const words = ['东赤道', '特里洛尼', '罗德里格岛'];
  inputs.forEach((inp, idx) => {
    inp.value = words[idx];
    inp.dispatchEvent(new Event('input', { bubbles: true }));
    inp.dispatchEvent(new Event('change', { bubbles: true }));
  });
  return Array.from(inputs).map(inp => inp.value);
}
```

## 修复方案

**选项 1：** 修改输入框的 event listener，确保 `fill` 操作触发的事件能被正确处理

**选项 2：** 文档化这个限制，要求自动化测试使用 JS 方式设置值

**选项 3：** 优化 OpenClaw 的 `fill` 工具，自动触发必要的事件（`input` + `change` + `keyup`）

## 影响范围

- 只影响自动化测试

- 可能其他输入框也有同样的问题
