## 核心创新 #1：本地语言助记词系统

### 问题背景

传统 BIP39 助记词只支持英文：
```
abandon, ability, able, about, above, absent, absorb, ...
```

对于中文、日文、韩文用户来说，这些词毫无意义，容易导致：
- 记忆困难（用户无法用母语理解助记词）
- 备份风险（用户抄写时容易出错）
- 安全隐患（用户倾向于截图或拍照备份，增加泄露风险）

### 解决方案：地名映射词库

**核心思想：** 将 BIP39 的 2048 个词表映射到**各语言的 2048 个常见地名或单字**，保持密钥派生兼容性。

### 实现细节

#### 第一步：构建多语言词库

每种支持的语言都有一个 2048-词表，基于**人类熟悉度**精心选择：

**中文（Simplified Chinese）：**
```json
[
  0: "北京",
  1: "上海",
  2: "广州",
  3: "深圳",
  4: "成都",
  5: "重庆",
  6: "杭州",
  7: "武汉",
  8: "西安",
  9: "南京",
  10: "天津",
  ...
  2047: "昌吉"  // 最后一个词
]
```

**日文（Japanese）：**
```json
[
  0: "あ",    // あ行第一个
  1: "い",
  2: "う",
  3: "え",
  4: "お",
  5: "か",
  ...
  2047: "ん"  // 最后的ん
]
```

**英文（English，BIP39 标准）：**
```json
[
  0: "abandon",
  1: "ability",
  2: "able",
  ...
  2047: "zone"
]
```

#### 第二步：BIP39 兼容的密钥派生

用户创建钱包时的流程：

```
1. 用户选择语言（例如中文）
   ↓
2. 生成 12-24 个单词的英文 BIP39 助记词
   例如：["abandon", "ability", "able", "about", "above", "absent", ...]
   ↓
3. 查表转换为本地语言
   例如：["北京", "上海", "广州", "深圳", "成都", "重庆", ...]
   ↓
4. 显示给用户（用户看到的是中文地名）
   "请抄写：北京 上海 广州 深圳 成都 重庆..."
   ↓
5. 用户输入验证时，接收中文地名
   用户输入："北京 上海 广州"
   ↓
6. 反向转换为英文 BIP39
   ["abandon", "ability", "able"]
   ↓
7. 调用 ethers.js 进行标准 BIP39 派生
   ethers.js.Wallet.fromMnemonic("abandon ability able...")
   ↓
8. 生成私钥、地址等（与标准 BIP39 完全兼容）
```

**关键优势：**
- ✅ **用户体验** — 用母语记忆和备份助记词
- ✅ **密钥兼容** — 仍然使用标准 BIP39 派生，可在任何 BIP39 钱包恢复
- ✅ **文化友好** — 地名符合各地文化，更易于口头传达

## 核心创新 #2：万语地址系统（简化版）

### 问题背景

钱包地址是冷硬的十六进制或 Base58 字符：
```
Ethereum: 0x742d35Cc6634C0532925a3b844Bc19e0A64d8d61
Tron:     TQb56CsxDNRVwMqe8d2AqN8P8dZpFvhXgk
Bitcoin:  1A1z7agoat2r9WarnL37eMkD8vYamdL5Z
```

问题：
- 用户无法记忆或口头传达
- 转账时容易出错（一个字符错误就会资产永久丢失）
- 地址验证依赖视觉逐字对比，容易被欺骗

### 解决方案：用地名/单字替代地址中间

**格式：**
```
[8 位随机数字] - [10 个本地语言单词] - [8 位随机数字]
```

**示例：**

```
标准以太坊地址：
0x742d35Cc6634C0532925a3b844Bc19e0A64d8d61

万语地址（中文）：
12345678-北京上海广州深圳成都重庆杭州武汉西安南京-87654321

万语地址（日文）：
24681357-あいうえおかきくけこ-13579246

万语地址（英文）：
11111111-abandon-ability-able-about-above-absent-absorb-abstract-absurd-abuse-22222222

万语地址（混合）：
99999999-北京-London-广州-Tokyo-成都-Paris-杭州-Moscow-西安-Dubai-11111111
```

### 设计思路

#### 1. 为什么要前后各 8 位数字？

- **安全性**：防止用户只使用地名部分（容易碰撞）
- **唯一性**：8 位数字提供 10^8 = 1 亿种组合，足以确保地址唯一
- **隐私**：数字部分保留加密的随机性，用户记不住

#### 2. 为什么是 10 个单词？

- **记忆力平衡**：10 个单词 = 40-50 bits 熵，足够安全，用户仍可记住
- **口头传达**：10 个单词可以在 20-30 秒内口头传达完整地址
- **错误检测**：如果 10 个单词中有任何一个错误，接收方可立即发现

#### 3. 单词来自哪里？

从 BIP39 词库的**对应索引位**：

```
地址末尾字节 #1：0x1A → 十进制 26 → BIP39 词表 #26 = "abandon"
地址末尾字节 #2：0x2B → 十进制 43 → BIP39 词表 #43 = "above"
...
（对中文用户，查中文词库对应位：#26 = "北京", #43 = "广州"）
```

### 实现流程

```javascript
// 伪代码
function generateWWAddress(baseAddress, language) {
  // 1. 提取地址末尾 10 字节（作为种子）
  const seed = baseAddress.slice(-20); // 最后 10 字节 = 20 个十六进制字符
  
  // 2. 生成前缀数字（8 位随机）
  const prefix = Math.floor(Math.random() * 100000000).toString().padStart(8, '0');
  
  // 3. 生成后缀数字（8 位随机）
  const suffix = Math.floor(Math.random() * 100000000).toString().padStart(8, '0');
  
  // 4. 将 seed 转换为 10 个词索引
  const indices = [];
  for (let i = 0; i < 10; i++) {
    // 每个字节提取 4 bits，映射到 0-2047
    const byte = parseInt(seed.substr(i * 2, 2), 16);
    const index = byte % 2048; // 确保在词库范围内
    indices.push(index);
  }
  
  // 5. 查表获取本地语言单词
  const wordlist = getWordlistFor(language); // 获取该语言的 2048-词表
  const words = indices.map(idx => wordlist[idx]);
  
  // 6. 拼装万语地址
  const wwAddress = `${prefix}-${words.join('')}-${suffix}`;
  
  return wwAddress;
}

// 示例调用
const ethAddress = "0x742d35Cc6634C0532925a3b844Bc19e0A64d8d61";
const chineseWWAddress = generateWWAddress(ethAddress, "zh");
// 输出：12345678-北京上海广州深圳成都重庆杭州武汉西安南京-87654321
```

### 用户体验改进

**场景 1：钱包之间转账**

```
发送方操作流程：
1. 点击"发送"
2. 输入接收方地址（两种方式）
   a. 粘贴标准地址：0x742d35Cc...
   b. 粘贴万语地址：12345678-北京上海广州深圳成都重庆杭州武汉西安南京-87654321
3. 系统自动识别并转换
4. 显示转换结果供用户确认

接收方验证流程：
1. 告诉发送方："把钱发到我的万语地址"
2. 发送方读出："12345678-北京上海广州深圳成都重庆杭州武汉西安南京-87654321"
3. 接收方确认："前缀是 12345678，中间是北京上海广州...，后缀是 87654321"
4. 接收方点击确认 ✓
5. 完成！
```

**对比传统方式的改进：**

| 操作 | 传统地址 | 万语地址 |
|------|--------|--------|
| 口头传达 | 不可能（40 个字符） | ✅ 可行（10 个词） |
| 视觉确认 | 容易出错 | ✅ 用词义识别，更清晰 |
| 防钓鱼 | 难以辨别假地址 | ✅ 用词的一致性检查 |
| 记忆 | 不可能 | ✅ 用户可记住 |
| 短信/语音传输 | 易出错 | ✅ 更易验证 |

## 核心创新 #3：本地密钥派生

### 关键点

**所有密钥派生都发生在用户设备本地：**

```
用户设备（浏览器）
  ├─ 用户选择语言 → 本地生成 BIP39 英文助记词（ethers.js）
  ├─ 转换为该语言的地名词表
  ├─ 用户备份（看到的是中文地名）
  ├─ 用户输入验证（输入中文地名）
  ├─ 反向转换为英文 BIP39
  ├─ 派生私钥（ethers.js.Wallet.fromMnemonic）
  ├─ 派生各链地址（Ethereum, Tron, Bitcoin）
  └─ 加密存储在 localStorage（仅本地，不上传）

服务器
  └─ 只接收已签名的交易，永不接收私钥或明文数据
```

## 总结

WorldToken 的技术创新在于：

1. **本地化不妥协**：用母语助记词和地址，而不是英文翻译
2. **安全不牺牲**：保持与 BIP39 的完全兼容，不创造新的密钥标准
3. **易用性突破**：地址从 40 个字符压缩到 10 个单词，可口头传达
4. **本地优先**：所有关键操作在用户设备本地进行，服务器零信任

这三个创新结合在一起，为全球 120+ 种语言的用户提供了一个真正的**本地化、安全、易用的加密钱包**。
